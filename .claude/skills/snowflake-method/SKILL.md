# 雪花写作法 — 10步写出一篇好小说

## 核心理念

雪花写作法的核心是**从小到大、逐步扩展**——就像雪花分形一样，从一个简单的点出发，每一步都在上一步的基础上增加细节和复杂度。不要试图一步到位，而是通过10个渐进步骤，让故事自然生长。

**关键原则：**
- 每一步都建立在前一步的基础上
- 随时可以回到前面的步骤进行修改
- 步骤之间可以灵活调整，不必严格按顺序
- 人物和情节同等重要，交替深化

## 工作流程

### 启动项目

当用户开始一个新的小说创作时：

1. 询问用户的故事想法、类型（genre）、主题
2. 在当前目录下创建项目文件夹结构：

```
{项目名}/
├── 01-一句话概括.md
├── 02-一段式概括.md
├── 03-人物简介.md
├── 04-故事梗概.md
├── 05-人物小传.md
├── 06-完整大纲.md
├── 07-人物宝典.md
├── 08-场景清单.md
├── 09-场景描写/
│   ├── 场景01.md
│   ├── 场景02.md
│   └── ...
└── 项目状态.md
```

3. 使用 `templates/project-status.md` 创建项目状态追踪文件
4. 按步骤引导用户完成创作

### 步骤一：一句话概括

**目标：** 用一句话（不超过25个字）概括整个故事。

**规则：**
- 不使用角色名字，用"一个XX的人"替代
- 聚焦宏观全局，不纠缠细节
- 抓住故事的核心矛盾和吸引力
- 格式：一个[什么样的人]+遇到/面对[什么问题]+[怎样的结局/抉择]

**引导方式：**
1. 先问用户想写什么样的故事
2. 帮助用户提炼核心概念
3. 与用户一起打磨，直到满意
4. 将结果保存到 `01-一句话概括.md`（使用模板 `templates/step1-one-sentence.md`）

**示例：** "一个身怀绝技的物理学家必须拯救他的妻子免遭恐怖分子的毒手。"（《虎胆龙威》）

### 步骤二：一段式概括

**目标：** 将一句话扩展为一段话（约五句），构建三幕结构。

**五句话结构：**
1. **背景铺垫**：故事的起因和背景
2. **第一次灾难**：第一幕结尾，主角主动行动导致的困境
3. **第二次灾难**：第二幕中段，主角再次陷入更大的困境
4. **第三次灾难**：第二幕结尾，最大的危机/黑暗时刻
5. **结局**：第三幕，最终的解决

**引导方式：**
1. 基于步骤一的一句话概括进行扩展
2. 确保每个灾难都比上一个更严重
3. 前两次灾难由外部原因引起，第三次灾难是主角自身问题导致
4. 结局要从灾难的逻辑中自然生长
5. 保存到 `02-一段式概括.md`（使用模板 `templates/step2-one-paragraph.md`）

### 步骤三：人物简介

**目标：** 为每个主要角色创建简明摘要。

**每个角色需要：**
- **名字**
- **角色定位**（主角/反派/盟友/爱情线等）
- **一句话概括**（角色故事线）
- **动力/价值观**：对这个人来说，没有什么比___更重要（抽象层面）
- **目标**：具体想要达成什么（具体层面）
- **冲突**：什么阻碍他/她达成目标
- **顿悟**：故事结束时他/她领悟了什么
- **一段话概括**：角色在故事中的完整弧线

**引导方式：**
1. 先确定有几个主要角色（通常3-5个）
2. 对每个角色逐一填写上述信息
3. 确保角色之间的冲突互相关联
4. 确保角色的价值观和目标存在内在张力
5. 保存到 `03-人物简介.md`（使用模板 `templates/step3-character-summary.md`）

### 步骤四：完善故事梗概

**目标：** 将步骤二的每句话扩展为一段，形成约一页的故事梗概。

**规则：**
- 每段话结尾应是一个"灾难"或转折点（前三段）
- 最后一段是故事结局
- 此时可能发现步骤一到三的问题，可以回去修改
- 这是创意探索的过程，允许反复修改

**保存到** `04-故事梗概.md`（使用模板 `templates/step4-synopsis.md`）

### 步骤五：人物小传

**目标：** 为每个主要角色写一页的人物小传，次要角色写半页。

**要求：**
- 从角色自身的视角来写
- 包含角色的背景故事
- 描述角色在故事中的完整弧线
- 展示角色的内心世界和转变
- 与主线情节保持一致

**引导方式：**
1. 基于步骤三的信息扩展
2. 用第一人称或紧贴角色的第三人称视角
3. 写完后检查是否需要回去修改前面的步骤
4. 保存到 `05-人物小传.md`（使用模板 `templates/step5-character-synopsis.md`）

### 步骤六：完整大纲

**目标：** 将步骤四的一页梗概扩展为约四页的完整大纲。

**方法：**
- 将每一段扩展为一整页
- 增加子情节和更多细节
- 在大纲中标注关键的情节转折
- 确保因果关系链清晰
- 注意节奏的变化

**保存到** `06-完整大纲.md`（使用模板 `templates/step6-full-outline.md`）

### 步骤七：人物宝典

**目标：** 为每个角色创建完整的人物档案。

**包含信息：**
- 基本信息（年龄、外貌、职业、家庭等）
- 性格特征和习惯
- 背景故事（详细版）
- 内心冲突和外部冲突
- 在故事中的变化弧线
- 与其他角色的关系
- 关键场景中的表现
- 说话方式和口头禅

**保存到** `07-人物宝典.md`（使用模板 `templates/step7-character-bible.md`）

### 步骤八：场景清单

**目标：** 列出所有场景的清单表格。

**每个场景包含：**
| 场景编号 | 视角人物 | 场景概要 | 预估页数 | 所属章节 |
|---------|---------|---------|---------|---------|

**引导方式：**
1. 从步骤六的大纲中提取所有场景
2. 为每个场景确定视角人物
3. 用一两句话概括场景内容
4. 估算每个场景的篇幅
5. 保存到 `08-场景清单.md`（使用模板 `templates/step8-scene-list.md`）

### 步骤九：场景描写

**目标：** 为每个场景写出详细的叙事描写。

**场景分为两种类型：**

**主动场景（Proactive）：**
- **目标（Goal）**：视角人物想要达成什么
- **冲突（Conflict）**：什么阻碍了他/她
- **挫折（Setback）**：场景结束时情况变得更糟

**被动场景（Reactive）：**
- **反应（Reaction）**：角色对上一个场景挫折的情感反应
- **困境（Dilemma）**：面临的两难选择
- **决定（Decision）**：做出选择，引向下一个主动场景

**引导方式：**
1. 主动场景和被动场景交替出现
2. 每个场景写几段话的详细描写
3. 确保场景之间因果链清晰
4. 为每个场景创建单独文件，保存到 `09-场景描写/` 目录
5. 使用模板 `templates/step9-scene.md`

### 步骤十：写小说

**目标：** 基于前面所有准备工作，开始写第一稿。

**要点：**
- 有了前九步的准备，写第一稿会快得多
- 不必追求完美，第一稿的目的是把故事写出来
- 遇到问题可以回到前面的步骤修改设计
- 写作过程中可能会发现新的灵感，拥抱它们

**操作方式：**
1. 建议用户根据场景描写逐场景写作
2. Claude 可以协助扩展场景描写为完整的叙事文本
3. 帮助用户保持一致的语言风格和叙事节奏
4. 定期更新项目状态文件

## 互动原则

1. **协作式引导**：每完成一个步骤都与用户确认，再进入下一步
2. **灵活跳转**：允许用户跳到任意步骤或回退修改
3. **迭代改进**：鼓励用户反复打磨，不急于推进
4. **记录进度**：每次完成/修改一步都更新 `项目状态.md`
5. **保持一致性**：修改任何步骤时提醒用户检查受影响的后续步骤
6. **中文为主**：所有输出内容使用中文

## 继续已有项目

当用户要继续一个已有项目时：
1. 读取 `项目状态.md` 了解当前进度
2. 读取最近完成的步骤文件
3. 从下一个待完成步骤继续引导
